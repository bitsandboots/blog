Quick howto for the Hugo site generator

Wed, 25 Oct 2017 by Matthew in  computing, linux, hugo, git

This documents the various steps I took to make this blog. I use Slackware Linux, other systems may operate differently.

Install Hugo

There isn’t a recent build available specifically for Slackware, so I retrieved the generic AMD64 version and copied it to my ~/bin directory. Golang builds self contained executables so there are no dependencies to install.

Create a site:

hugo new site bitsandboots

Install and configure a theme

Install a theme

cd bitsandboots/themes
git clone --depth=1 https://github.com/alanorth/hugo-theme-bootstrap4-blog.git

Copy the hugo-theme-bootstrap4-blog/exampleSite/config.toml to the sites root and modify it.

  • sharingicons = false
  • date_format = "Mon, 02 Feb 2006"
  • `title = “Bits and Boots”

Make some content

The content is formatted with Markdown, but before that we have Front Matter:

This is the front matter for this page:

+++
title = "Quick howto for the Hugo site generator"
date = "2017-10-25"
draft = "false"
tags = ["computing","linux","hugo","git"]
+++

Content Suggestions

These come from the designer of this theme:

  • Keep blog posts in the content/post directory, for example: content/post/my-first-post.md
  • Keep static pages in the content directory, for example: content/about.md
  • Keep media like images in the static directory, for example: static/2016/10/screenshot.png

Host the site on Github Pages

Some of these steps are taken from the Hugo web site.

Create an account

Complete the Hello world tutorial

Create a repository

Add a line to config.toml:

enableGitInfo = true

initialise the folder as a git repo

git init

git config user.email "you@example.com"
git config user.name "bitsandboots"     

git remote add origin https://github.com/bitsandboots/blog.git
git push -u origin gh-pages
